Network Logging Lab: Linux Network Tools Exploration

Lab Objective

In this lab, you'll learn to use three essential Linux network tools: nmap, tcpdump, and netcat. These tools are crucial for network exploration, packet capture, and network communication.

Prerequisites

  • Linux environment (Ubuntu, CentOS, or Kali recommended)
  • Root or sudo access
  • Basic understanding of networking concepts
Important: Always ensure you have permission before scanning or probing networks you do not own.

Step 1: Installation

First, install the required tools:

sudo apt-get update sudo apt-get install nmap tcpdump netcat

Step 2: Network Scanning with nmap

Nmap (Network Mapper) is a powerful tool for network discovery and security auditing.

Basic Scan

nmap localhost

Detailed Scan

sudo nmap -sV -p- localhost
Tip: The -sV flag detects service/version info, -p- scans all ports.

Step 3: Packet Capture with tcpdump

Tcpdump allows you to capture and analyze network traffic in real-time.

Capture Basic Traffic

sudo tcpdump -i eth0 -n

Capture Specific Protocol

sudo tcpdump -i eth0 tcp port 80
Note: Replace eth0 with your network interface name.

Step 4: Network Communication with netcat

Netcat is a networking utility for reading from and writing to network connections.

Setting Up a Listener

nc -l -p 12345

Connecting to a Listener

nc localhost 12345
Example: This creates a simple chat-like communication between two terminals.

Lab Conclusion

You've now been introduced to three fundamental network tools in Linux. These tools are essential for network administrators, security professionals, and developers for understanding and managing network communications.
